Chapter one — start here

Ten moves, ordered by impact over effort

If only one PR comes out of this report, make it #1 — it closes the only live PHI exposures and is almost entirely deletion.

01

Strip PHI from debug output

Remove the transcript-leaking print() lines and the OSLog text= fragment (SEC-1, SEC-2). ~90 mechanical deletions; the safe AppLogger calls already exist beside them.

S · High
02

Wire up PHI clearing on quit + idle

Call sessionStore.clear() from a willTerminate observer and drive the already-written checkIdleTimeout() from something real (SEC-4, SEC-5).

S–M · High
03

Flip AppLogger's privacy default

Wrappers currently force privacy: .public on every message — the root cause behind finding #1. Default the dynamic message to .private (SEC-3).

M · High
04

Fix the wake-word reconfiguration race

The one Critical: updateKeywords tears down the spotter across an await while frames keep arriving. Single-flight guard + ordered frame consumer fixes both CON-1 and CON-2.

M · Critical
05

Repair the two custom SwiftLint rules

One doesn't exist, the other is configured to match only comments. The concurrency guardrails the docs promise are not actually running (TST-1, TST-2).

S–M · High
06

Stop re-hashing 5.2 GB on every launch

First Generate-Notes of each launch streams and hashes the full model file even when intact. Persist a verified receipt keyed on size/mtime/revision (PRF-3).

M · High
07

Debounce settings writes, delete synchronize()

A slider drag is ~10–20 full-blob synchronous disk flushes (PRF-4). Smallest effort-to-win ratio in the performance set.

The ASR engine tracks branch: "main" — flagged independently by three reviewers. Pin to a tag or revision (ARC-2).

S · High

Conceal the clipboard export path

The SOAP-note copy fallback writes PHI to NSPasteboard with no concealed/transient type and no expiry — it outlives every other clearing mechanism (SEC-7).

Chapter two — security & PHI

Security & PHI

No committed secrets, no plaintext credentials, no non-TLS endpoints, and the Cliniko client's PHI discipline is genuinely good. The gaps are at the edges: debug logging, lifecycle clearing that was designed but never wired, and supply-chain hygiene in scripts and CI.

Chapter three — concurrency

Concurrency & correctness

The dreaded @Observable + actor-existential crash pattern is absent everywhere — the house rules are working. The real findings cluster in the wake-word path, plus a set of latent actor-reentrancy races currently masked by single-caller discipline in AppState.

Chapter four — performance

Performance & energy

Two themes: multi-gigabyte ML models that are loaded carefully but never let go, and an always-on menu-bar app doing per-frame work it doesn't need to — eager log strings, polling timers, and whole-tree SwiftUI invalidation from audio-rate property writes.

Chapter five — architecture

Architecture & enhancements

The service layer, error taxonomy, and type-safety patterns are strong. The debt is concentrated in one god object, ~2,600 lines of confirmed dead UI, and a set of user-facing gaps — localization, VoiceOver, software updates, and clinical onboarding — that matter for an app heading toward clinicians.

Chapter six — testing & CI

Testing, CI & tooling

The test infrastructure is unusually sophisticated — but the two custom lint rules the docs lean on aren't actually enforcing anything, a few PHI/HTTP-bearing services lack direct tests, and several shell scripts have sharp edges.

Chapter seven — credit where due

What this codebase gets right

All five reviewers independently called out strengths. These patterns are worth protecting in review — several are better than what most production Swift apps ship.

Methodology

Five independent review passes ran in parallel on 12 June 2026, each reading source directly (not just grepping), each scoped to one dimension and primed with the project's own hard rules from AGENTS.md and .claude/references/:

  • Security & PHI — Cliniko layer, Keychain, logging tree, model-download integrity, scripts, CI, entitlements
  • Concurrency — actors, @Observable, audio-thread callbacks, nonisolated(unsafe), task lifecycle
  • Performance — render loops, model residency, polling timers, audio pipeline, persistence I/O
  • Architecture — layering, dead code, type safety, error surfacing, UX/a11y/i18n enhancement opportunities
  • Testing & CI — coverage gaps, test quality, workflow config, lint rules, shell scripts

Findings were deduplicated across reviewers; where two or more flagged the same issue independently it is noted on the card. Locked technical decisions (MLX runtime, Gemma 4, session-only persistence, Cliniko scope, testing stack) were excluded from scope — the one finding that touches a locked decision (PRF-2, model eviction) is explicitly marked as needing product sign-off. Every finding cites file:line and was confirmed in surrounding code; severities are the reviewers' calibrated ratings, not raw counts.
